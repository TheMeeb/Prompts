You are a senior full-stack software engineer with expertise in designing secure, scalable, and production-ready web applications.

Design and implement a full-stack e-commerce web application.

Requirements:

Frontend:
- Use HTML, CSS, and modern JavaScript (ES6+) or a frontend framework (optional)
- Build responsive UI for:
  - Product listing page
  - Product detail page
  - Search functionality
  - Shopping cart
  - User authentication (login/signup)
  - Checkout flow
  - Admin dashboard
- Communicate with backend via REST APIs

Backend:
- Use Node.js with Express
- Use SQLite (or similar relational database)
- Implement RESTful APIs for:
  - User authentication
  - Product management
  - Search functionality
  - Cart and order management
  - Admin operations

Features:

User Features:
- User registration and login
- Browse products
- Search products by name/category
- Add/remove items from cart
- Place orders (simulate payment)

Admin Features:
- Admin login
- Add/edit/delete products
- View orders
- Manage users

Search:
- Implement search with filtering (by name, category, price range)
- Ensure efficient and secure query handling

Security (CRITICAL):

Authentication & Authorization:
- Use secure authentication (JWT or session-based)
- Implement role-based access control (RBAC)
- Restrict admin routes to authorized users only
- Prevent privilege escalation

Input Handling:
- Validate and sanitize all user inputs
- Prevent SQL injection using parameterized queries or ORM
- Prevent Cross-Site Scripting (XSS) in product names, descriptions, and user input
- Prevent Cross-Site Request Forgery (CSRF) if using cookies

E-commerce Specific:
- Do not trust client-side price or cart data
- Validate product prices and quantities on the server
- Prevent manipulation of order totals
- Ensure secure handling of checkout logic

General Security:
- Hash passwords securely (bcrypt)
- Use environment variables for secrets
- Implement rate limiting on sensitive endpoints (login, checkout)
- Handle errors securely without exposing stack traces

Database Design:
- users (id, username, email, password, role)
- products (id, name, description, price, stock)
- orders (id, user_id, total_price, created_at)
- order_items (id, order_id, product_id, quantity)

Architecture:
- Follow modular or MVC architecture
- Separate routes, controllers, services, and database logic
- Use middleware for authentication, authorization, and validation

Output:
- Provide complete project folder structure
- Provide all frontend and backend code
- Include database schema and sample data
- Clearly explain how authentication, authorization, and checkout security are implemented
- Provide step-by-step instructions to run the application
